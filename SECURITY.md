# Security Policy

## Supported versions
This repository currently tracks active development on the default branch.

## Reporting a vulnerability
Please do not open public issues for security vulnerabilities.

Send a private report with:
- Affected component/module
- Reproduction steps
- Impact assessment
- Suggested remediation (if available)

Until a dedicated security contact is published, coordinate with repository maintainers directly.

## Secret handling reminders
- Never commit production database credentials.
- Never commit production encryption secrets.
- Rotate secrets immediately if exposure is suspected.
